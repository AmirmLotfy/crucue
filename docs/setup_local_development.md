# Local Development Setup

## Prerequisites

| Tool | Version | Notes |
|------|---------|-------|
| Flutter | 3.22+ | `flutter --version` |
| Dart | 3.4+ | bundled with Flutter |
| Node.js | 20+ | for Cloud Functions |
| Firebase CLI | latest | `npm install -g firebase-tools` |
| Xcode | 15+ | macOS only, for iOS builds |
| Android Studio | Hedgehog+ | for Android builds |

## 1. Clone the repository

```bash
git clone <repo-url>
cd crucue
```

```bash

```

## 3. Firebase platform configuration

The Firebase config files are gitignored and must be added manually.

### Android

1. In [Firebase Console](https://console.firebase.google.com), open your project.
2. Go to **Project Settings → Your apps → Android app** (`com.crucue.app`).
3. Download `google-services.json`.
4. Place it at `android/app/google-services.json`.

### iOS

1. In Firebase Console, go to **Project Settings → Your apps → iOS app** (`com.crucue.app`).
2. Download `GoogleService-Info.plist`.
3. Place it at `ios/Runner/GoogleService-Info.plist`.
4. **iOS 15.0+** deployment target (see `ios/Podfile` and Xcode `IPHONEOS_DEPLOYMENT_TARGET`) — required for **Firebase iOS SDK 12.x** bundled with FlutterFire BoM 4.12. After `flutter pub get`, run `cd ios && pod install` with **full Xcode** as the active developer directory (`sudo xcode-select -s /Applications/Xcode.app/Contents/Developer`), not Command Line Tools only.

### Dart (FlutterFire CLI)

The `lib/firebase_options.dart` file is generated by FlutterFire and is already present in the repo. If you are using a different Firebase project, regenerate it:

```bash
dart pub global activate flutterfire_cli
flutterfire configure
```

## 4. Cloud Functions setup

```bash
cd functions
npm install
```

### Gemma 4 API key

All AI calls require a Gemma 4 API key. This is **never** put in the Flutter app.

For local Functions development:
```bash
# Create functions/.env from the example
cp functions/.env.example functions/.env
# Edit functions/.env and set your GEMMA4_API_KEY
```

For production:
```bash
firebase functions:secrets:set GEMMA4_API_KEY
```

Get a Gemma 4 API key from [Google AI Studio](https://aistudio.google.com/).

## 5. Running the app

```bash
# Run on connected device or simulator
flutter run

# Run on specific device
flutter run -d <device-id>

# List available devices
flutter devices
```

### Firebase local emulators (optional)

For offline development, use the Firebase Emulator Suite:

```bash
# Install emulators if needed
firebase init emulators

# Start Auth + Firestore + Functions emulators
firebase emulators:start --only auth,firestore,functions
```

Then update `lib/firebase_options.dart` or use environment flags to point to the emulator URLs.

## Common errors

### `google-services.json not found`
The file is gitignored. Download from Firebase Console and place at `android/app/google-services.json`.

### `MissingPluginException` for record/audio
Ensure you are running on a physical device or a simulator that supports audio. Some simulators do not expose a microphone.

### Cloud Functions: `GEMMA4_API_KEY is not set`
Add the key to `functions/.env` for local development. See step 4.

### `flutter analyze` reports errors
Run `flutter pub get` first. If errors persist, check the CHANGELOG for any breaking changes to the `AiEngine` interface.

### Dark mode / theme issues
All screens use `Theme.of(context).colorScheme.*` and `CrucueTokens`. If you are adding new UI, do not use `Color(0xff...)` literals. See [`docs/theme_system.md`](theme_system.md).

## Project structure quick reference

```
lib/
  app/              # providers.dart, constants.dart, router.dart
  core/ai/          # AiEngine, RemoteGemma4Engine, on-device stubs, registry
  core/audio/       # Recorder, TTS, playback services, tts_provider
  core/config/      # FeatureFlags, EnvConfig
  core/design/      # Shared widgets (AppButton, AppInput, etc.)
  core/logic/       # helper_methods (navigation), cache_helper, notifications
  core/observability/ # CrucueAnalytics typed events
  core/services/    # FirestoreService, StorageService, CloudFunctionsService
  core/theme.dart   # CrucueTokens + AppTheme.light/dark
  features/         # Feature modules by domain
  shared/models/    # 9 typed Dart models
  views/            # Auth, home, chat, results, settings, legal screens
```
