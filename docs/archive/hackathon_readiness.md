# Hackathon Readiness — Crucue

## Primary Use Case

**A caregiver who just had a difficult moment with their loved one logs it (by voice or text) and gets a calm, structured, personalized support plan in under 30 seconds.**

This is the demo. Everything else is supporting context.

---

## Hero Flow

```
Home
  ↓ (tap profile or FAB)
Profile Detail
  ↓ (Log Challenge button or voice note chip)
Add Incident Screen
  ↓ (voice) → Voice Recording → Processing → Transcript Review
  ↓ (text) → Fill form → Save
    ↓
Results View (Support Plan)
  — Generated by Gemma 4 (gemma-4-26b-a4b-it) via Cloud Functions
  — Structured JSON output: summary, steps, message draft, follow-up
  — TTS playback button (listen to the plan)
  — Save to profile
    ↓
Chat View (Grounded follow-up)
  — Grounded in the plan and profile context
  — Voice input available
    ↓
Reflection / Check-In
  — Rate what worked, what didn't
  — Add voice notes
  — Save as routine if helpful
```

**All four broken links in this flow were fixed in the readiness pass.**

---

## Gemma 4 Visibility

- Model: `gemma-4-26b-a4b-it` (Mixture-of-Experts, production default)
- All plan generation, chat, insights, and voice extraction use Gemma 4
- Cloud Functions call the Gemma 4 API with structured JSON schemas
- On-device path scaffolded for future demo: `gemma-4-e2b-it` / `gemma-4-e4b-it`
- Settings → AI Engine shows the mode selector to users

---

## Structured Outputs

Every AI call uses a JSON schema:
- `SUPPORT_PLAN_SCHEMA` — summary, steps, messageDraft, safetyNote, followUpQuestions
- `CHAT_RESPONSE_SCHEMA` — message, suggestedFollowUps, safetyFlag
- `INSIGHT_SUMMARY_SCHEMA` — summary, patterns, whatWorked, suggestions, moodTrend
- `ROUTINE_SUGGESTION_SCHEMA` — title, steps, frequency, tags, rationale

## What Crucue is NOT (for demo honesty)

- Not a licensed medical or therapeutic service (stated in-app)
- Not doing client-side AI — all inference is server-side via Cloud Functions
- Not offline yet — on-device AI is scaffolded but model weights not deployed
- Not storing any voice audio beyond processing — deleted after transcription
